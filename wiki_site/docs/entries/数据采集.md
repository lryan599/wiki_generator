压铸数据采集是指针对压铸生产过程，通过传感器、控制器、数据记录器及配套软件，自动或半自动地收集并记录模具温度、熔汤温度、压射压力‑速度曲线、保压时间、冲头行程、合型力、真空度等工艺参数，以及铸件缺陷、设备运行状态、能耗等质量与管理数据的系统化活动[[S8,S45]]。其输出可直接用于实时过程监控、异常报警、质量追溯与统计过程控制，是压铸智能制造与全生命周期质量管理的基础[[S8,S24]]。

## 系统组成

典型的数据采集系统由现场感知硬件、信号预处理装置、控制与传输单元以及上位软件构成，可进一步抽象为分层架构。

### 典型硬件

**传感器**
压铸机各执行部位的物理量由多类传感器拾取，主力类别如下表所示[[S2,S16,S35]]。

| 类别 | 典型传感器 | 测量对象 |
|------|------------|----------|
| 温度 | 镍铬‑镍硅热电偶、铂铑‑铂热电偶、K型热电偶 | 合金液温度、模具温度、压室温度、冷却液温度 |
| 压力/力 | 荷重传感器、液压压力传感器 | 液压系统油压、冷却水压力、压射压力、合型力 |
| 位移/速度 | 电感位移传感器、脉冲编码器、挠性线‑鼓轮型位移传感器 | 炉内液面位置、浇包位置、冲头行程/运动速度 |
| 流量 | 流量传感器 | 冷却水流量 |
| 辅助 | 光电传感器、液位传感器 | 过程位置判定 |

**信号调理**
传感器输出的微弱模拟信号需经信号调理装置搭建测量电桥、提供激励电压，并进行放大与滤波，转换为后续模数转换模块可接受的标准信号[[S29,S2]]。

**控制器与采集模块**
- **PLC**：如西门子 S5-115U 等需额外配置定位模板与模拟量输入/输出模板，校验要点包括扫描速度、用户程序存储容量、I/O 点数与扩展能力、通讯模块支持性及特殊功能模块扩展能力[[S3,S39]]。
- **专用记录仪**：GB/T 37365‑2019 推荐采用无纸记录仪进行压铸单元性能检测，并连续记录 50 次运行周期的全量数据[[S39]]。多通道高速紫外线记录仪可同步采集冲头压力、位移、压射曲线等多组时序信号[[S47]]。
- **模块化采集卡**：如 NI USB‑9211 热电偶采集模块，可直接接入多组温度传感器并通过 USB 总线上传数据，适配模具多测点温度采集[[S22]]。

上述硬件的部署形式可分为移动式测试小车（可快速转移到多台压铸机进行工况标定）和机柜内置式（适配长期在线采集）[[S35]]。

### 典型软件与接口
现代压铸机控制系统普遍内嵌数据收集与分析功能，可显示压射曲线、顶出/合模速度曲线，记录每个循环的时间、位移、压力、速度、温控状态、故障类型等全流程数据，支持参数设定值与实际值的对比、偏差监控及质量统计报表输出[[S24,S25]]。上层系统常通过 SCADA、制造执行系统接口和集中式数据库实现跨设备整合。一种由奥迪与亚琛工业大学联合公开的架构采用 OPC‑UA 标准统一接入异构 PLC 和传感器，通过面向服务的分布式架构封装存量 SCADA/MES 系统，并利用 HTML5 轻量化 Web 看板实现实时可视化与流数据挖掘[[S31,S44]]。

### 典型分层架构
以面向薄壁件模温调控的系统为例，数据采集系统常划分为五层[[S36,S42]]：

![压铸数据采集系统五层总体架构图](http://192.168.150.150:8000/api/v1/workspaces/die_casting_wiki_v2/document-elements/d7a310dc-4e20-479c-9ff8-08df254107bc/resource)

- **设备层**：布置在压铸机关键部位的传感器及电磁阀、液压油缸、模温机等执行机构。
- **采集层**：完成原始物理信号的模数转换，输出标准数字信号。
- **控制层**：基于 PLC 比对采集数据与预设工艺参数，向液压、冷却等执行机构发送实时调节指令。
- **存储层**：采用关系型或非关系型数据库分类存储全量历史工艺数据。
- **应用层**：提供可视化报表、工艺监控、质量追溯、设备维护等上层服务。

## 关键采集参数与采样要求

### 核心参数维度
在一次压射循环中，系统需完整记录冲头位置（量程 0~1500 mm）、压射速度（量程 0~8 m/s）以及作用于合金的压力（量程 0~2000 bar）三条动态曲线，通过对比实际曲线与预设曲线的吻合度快速定位质量异常[[S45]]。全要素采集可覆盖 75 项与过程强相关的参数维度，涵盖位置、喷涂、计时、基础设置、注射设置等设备侧数据，以及短射、裂纹、起泡、变形、缩孔等 8 类铸件缺陷记录[[S8]]。人工辅助记录可落在标准化的“压铸工艺参数记录表”上，内容包含模具号、合金牌号、操作者、留模时间、余料饼厚度、合金温度、压射表压、压射速度、成品/废品数量及废品率等[[S26]]。

### 动态采样频率要求
冷室压铸机的压射过程属于毫秒级动态过程：从慢压射到充型结束、增压建压的全过程仅约 0.1 s，其中增压建压时间为 20~40 ms，内浇口充填时间甚至可低至 6 ms[[S18,S15]]。为此，GB/T 21269‑2023 明确规定压射力、最大空压射速度、建压时间和闭环控制速度误差的曲线记录仪采样时间不得大于 1 ms，即采样频率不低于 1000 Hz[[S4,S6]]。依据奈奎斯特采样定理，实际工程中为保证无失真还原信号，常在信号最高有效频率端每个周期采样 7~10 次[[S40,S46]]。以下为奈奎斯特采样定理的可视化示意，展示了不同采样间隔下正弦波的恢复效果及频率混叠现象[[S46]]：

![不同采样频率下的正弦波采样效果与混叠现象示意](http://192.168.150.150:8000/api/v1/workspaces/die_casting_wiki_v2/document-elements/ea92e0f6-ffa2-485d-a621-a20f32e021da/resource)

压铸机伺服调节响应精度需匹配 2~8 ms 的全动态响应时间，才能实现压射过程的实时闭环修正[[S34]]。商用多通道采集仪如 INV3060A，其采样频率覆盖 6.25~51.2 kHz，24 位 A/D 精度，可满足压射过程多参数同步高频采集的需求[[S14]]。

## 工艺作用

采集数据的直接价值体现在多种过程控制与优化场景中[[S8,S24]]。

**实时监控与报警**
通过连续比对实测值与该循环的设定允差，系统可在参数超限时触发分级报警（工作循环停止、卸荷、蜂鸣器报警、屏幕报警等），并向操作者或维护人员推送提示[[S11]]。

**统计过程控制**
对各工艺变量进行随机采样，计算均值与极差并绘制控制图，当绘图值超出预设控制界限时触发校正行动，实现过程的统计受控[[S7,S30]]。

**质量追溯与缺陷预防**
通过大数据平台采集 75 项工艺参数并进行相关性分析，已验证高速压射速度、低速行程、最大压射速度等参数与冷隔、气孔类缺陷强相关；在缺陷出现时，工程师可优先核查这些变量[[S8,S1]]。在线 X 射线检测模块可在铸件图像进入队列时自动推理并输出带缺陷分割结果的检测报告，离线模块则支持批量导入历史图像进行事后追溯[[S43]]。

**设备预防性维护**
基于单台设备的能耗、生产节拍、故障停机时长等数据计算设备整体有效运行率，同时利用油液温度、压力等历史采集数据建立预测模型，提前预警液压泵等核心部件故障[[S12,S1]]。

## 分类与部署形态

### 在线全时采集 / 离线抽样采集
在线采集模块自动识别新增的 X 射线检测图像并完成实时推理，适用于在线检测；离线模块支持批量导入历史图像，适用于事后质量追溯与产品抽检[[S43]]。在传统人工模式下，压铸工艺参数可通过标准记录表进行周期性手工填写与监控[[S26]]。

### 单机存储 / 中央数据库
单机存储形态将数据保存在与压铸机控制器直连的本地工控机中，不依赖外部网络，适合独立单机小批量生产。传统压铸机原生存储能力通常仅可缓存 50~140 条历史记录，超出后需人工导出备份[[S13,S11]]。中央数据库形态则将全车间多台压铸设备的采集数据统一上传至集中式企业级数据库，支持跨设备全局关联统计[[S21]]。

### 本地监控 / 远程云监控
本地监控将采集数据展示于车间侧的工作站显示屏，权限仅面向现场运维人员；远程云监控则通过 VPN 加密链路将数据同步至云端服务器，支持跨厂区的远程设备状态查看与异常排查[[S21]]。

## 相关标准与规范

**国际与行业标准**
北美压铸协会在产品规范中明确要求，模具温度、充型参数等核心过程变量必须纳入受控采集范围，通过全量记录支撑热传导规律管控，降低铸件内部孔隙率[[S28]]。压铸零件在提交批量生产审核（PPAP）时，必须同步提交全量过程控制与质量采集记录，作为强制审核材料[[S23]]。

**国家标准**
- GB/T 25747‑2022《镁合金压铸件》要求同批次产品必须留存外观质量、内部无损检测、化学成分和力学性能等全量取样检测采集记录，作为合格判定的法定支撑材料[[S10]]。
- GB/T 21269‑2023《冷室压铸机》规定动态压射性能参数采集的最低采样时间为 1 ms[[S4,S6]]。
- GB/T 37365‑2019《压铸单元 性能检测方法》推荐优先采用无纸记录仪进行数据采集与统计[[S39]]。

## 主要局限与挑战

**电磁干扰与信号噪声**
压铸现场存在大量大功率继电器、电磁阀频繁动作产生的电火花，导致强电磁干扰。采集系统需配置积分式 A/D 采样、工频整数倍采样窗口、去极值平均滤波等抗干扰措施，否则易出现信号畸变或数据丢失[[S20,S17]]。

**高速采集与海量存储成本**
以 48 通道、10 kHz 采样率、单次采集 90 s 为例，单批次数据量可达 82.4 MB；大批量连续生产下全生命周期存储成本极高。而传统压铸机原生存储能力通常仅能缓存 50~140 条历史记录，长期大数据分析需外接存储方案[[S5,S13,S11]]。

**异构协议集成困难**
不同品牌、不同代际的压铸设备普遍采用私有通信协议，PLC、传感器与控制系统之间缺乏统一数据交互标准，形成大量数据孤岛，跨厂商全场景统一采集难度大[[S27]]。

**数据安全与完整性风险**
传统工业控制系统在设计时几乎未考虑互联互通安全：依赖 DCOM 通信机制的经典 OPC 协议端口不固定，难以用常规防火墙防护；Windows 工控主机长期不打补丁、不安装杀毒软件；不同安全域之间缺乏有效隔离，采集数据被篡改或采集链路被恶意中断的风险严峻[[S38]]。当通过 Web 系统向供应链开放生产数据时，还需额外部署网络安全方案以防止敏感工艺数据泄露[[S37]]。
